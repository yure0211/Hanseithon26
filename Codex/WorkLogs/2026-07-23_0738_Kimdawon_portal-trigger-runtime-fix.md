# 포탈 트리거 비동작 수정

## 원인

- 네트워크 플레이어 프리팹의 게임 씬 이름이 `InGame_1`로 고정되어 있었다.
- 현재 포탈 개발 씬인 `InGame_portaldev`에서는 네트워크 플레이어의 조작, Rigidbody2D와 Collider2D가 비활성화되어 포탈 트리거에 진입할 수 없었다.
- 현재 씬을 직접 실행하면 씬에 배치된 로컬 `Bunny`와 `Turtle`이 사용되지만, 기존 포탈은 `DualPlayNetworkPlayer`만 인식했다.

## 수정 내용

- 네트워크 플레이어가 설정된 기본 씬뿐 아니라 이름이 `InGame`으로 시작하는 모든 게임 씬에서 활성화되도록 수정했다.
- 네트워크 연결 중에는 기존 서버 검증과 RPC 기반 포탈 처리를 그대로 사용한다.
- 네트워크 연결 없이 현재 씬을 직접 실행했을 때는 로컬 토끼와 거북이도 포탈이 인식하도록 보완했다.
- 로컬 플레이어도 포탈 진입 시 조작, 물리와 충돌체를 잠그고 두 캐릭터가 모두 들어오면 검은색 페이드 후 대상 씬으로 이동한다.
- `InGame_portaldev`를 Build Settings에 추가해 네트워크 씬으로 로드할 수 있게 했다.

## 확인

- `Assembly-CSharp.csproj` 빌드 결과 오류 0개.
- 기존 Unity 패키지 참조 버전 경고 2개만 존재하며 이번 포탈 코드와 무관하다.
- 플레이 모드 네트워크 테스트는 팀에서 진행한다.

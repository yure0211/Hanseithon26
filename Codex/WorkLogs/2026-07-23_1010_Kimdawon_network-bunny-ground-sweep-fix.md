# Kimdawon 작업 내용 - 네트워크 토끼 바닥 관통 방지

## 문제

- 에디터의 일반 플레이에서는 토끼가 바닥에 정상 착지하지만, 호스트 또는 클라이언트로 네트워크가 시작되면 토끼가 바닥 아래로 추락함.
- 거북이는 같은 바닥과 정상 충돌하므로 타일맵 콜라이더 자체보다 네트워크 플레이어의 소유권 전환 시 물리 상태와 위치 동기화 구간을 우선 보강함.

## 변경

- 로컬 소유 플레이어의 `Rigidbody2D`와 `BoxCollider2D` 상태를 매 물리 프레임에 다시 보장함.
- 네트워크 토끼가 하강할 때 직전 위치와 현재 위치 사이를 BoxCast로 검사해, 한 프레임 사이에 Ground 콜라이더를 건너뛴 경우 접촉면 위로 복구함.
- 실제 바닥이 없는 구간은 BoxCast가 적중하지 않으므로 정상적으로 추락할 수 있음.
- 네트워크 플레이어 Rigidbody2D는 Continuous 충돌 검사를 유지함.

## 확인

- `dotnet build Assembly-CSharp.csproj --no-restore --verbosity quiet`: 오류 0개.
- 실제 호스트/클라이언트 동시 접속 동작 확인은 사용자 테스트가 필요함.

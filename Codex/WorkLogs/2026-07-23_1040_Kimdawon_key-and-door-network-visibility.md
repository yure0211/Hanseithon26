# Kimdawon 작업 내용 - 열쇠 및 문 네트워크 표시 동기화

## 문제

- 토끼 화면에서는 열쇠 획득과 열쇠돌 해제가 보이지만 거북이 화면에는 해당 상태가 반영되지 않음.
- 기존 네트워크 RPC는 존재하지만 씬 오브젝트 식별자가 전체 SiblingIndex에 의존해 실행 환경별 계층 차이에 취약함.

## 변경

- 열쇠와 열쇠돌의 안정 ID를 계층 이름과 정적 로컬 위치 조합으로 생성하도록 변경함.
- 토끼가 서버에 상호작용 ID와 함께 월드 좌표 및 씬 경로를 전달함.
- 서버에서 ID 검색에 실패하더라도 같은 씬의 해당 좌표 근처 컴포넌트를 찾아 검증하고 처리함.
- 열쇠 획득 ClientRpc는 ID 대신 서버가 확정한 좌표와 씬 경로를 사용해 모든 화면의 동일 열쇠를 비활성화함.
- 열쇠돌 해제는 서버가 확정한 중심점과 반경을 모든 화면에 전달하는 기존 방식을 유지하면서 대상 검색을 보강함.

## 기대 결과

- 토끼가 열쇠를 먹으면 거북이 화면에서도 동일 열쇠가 사라짐.
- 토끼가 열쇠돌을 해제하면 거북이 화면에서도 반경 내 열쇠돌들이 함께 사라짐.
- 같은 프리팹을 여러 개 배치해도 씬과 위치로 개별 대상을 구분함.

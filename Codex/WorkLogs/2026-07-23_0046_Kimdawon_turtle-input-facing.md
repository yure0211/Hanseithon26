# 거북이 입력 방향 바라보기 추가

## 작업 목적

탑다운 방식으로 이동하는 거북이가 현재 이동 입력 방향을 바라보도록 `TurtleController`에 전용 함수를 추가한다.

## 변경한 파일

- `Assets/Scripts/TurtleController.cs`
- `Codex/WorkLogs/2026-07-23_0046_Kimdawon_turtle-input-facing.md`

## 핵심 변경 내용

- `FaceInputDirection()` 함수를 추가했다.
- 입력 벡터와 기본 정면인 `Vector2.up` 사이의 각도를 계산해 Rigidbody2D의 Z 회전에 적용한다.
- 입력이 없으면 회전하지 않아 마지막으로 바라보던 방향을 유지한다.
- 좌우·상하뿐 아니라 두 방향을 동시에 입력하면 대각선 방향도 바라본다.

## 확인 또는 테스트 결과

- 코드상 정규화된 이동 입력을 기준으로 물리 이동 전에 방향 회전이 적용되는 흐름을 확인했다.
- 기존 이동, 속도 제한, Animator 갱신 코드는 변경하지 않았다.
- 현재 세션에는 Unity MCP 연결 도구가 없어 Unity 컴파일과 Play Mode 동작은 자동 확인하지 못했다.

## 남은 작업 및 주의 사항

- 구현은 거북이 원본 스프라이트가 위쪽을 바라보는 것을 기준으로 한다.
- 실제 스프라이트 정면이 반대로 보이면 기준 각도에 180도를 더해 조정해야 한다.
- 네트워크 상대 화면에서도 회전이 보여야 한다면 `NetworkTransform`의 Z 회전 동기화 설정이 별도로 필요하다.

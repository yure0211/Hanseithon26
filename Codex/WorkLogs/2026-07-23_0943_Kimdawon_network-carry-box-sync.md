# 거북이 운반 상자 네트워크 동기화

## 원인

- 운반 상자는 `NetworkObject`가 아닌 일반 씬 오브젝트라 거북이 소유자의 화면에서만 위치가 변경됐다.
- 기존 `TurtleCarrySkill`은 로컬 Rigidbody2D만 이동하고 다른 플레이어에게 들기 상태와 좌표를 보내지 않았다.

## 수정 내용

- 각 씬 상자를 계층 순서 기반의 동일한 네트워크 식별자로 등록했다.
- 거북이의 `E` 입력을 소유 플레이어에서 서버로 요청하도록 연결했다.
- 서버가 가까운 상자를 검증한 후 들기와 내려놓기 상태를 모든 클라이언트에 전달한다.
- 들고 있는 동안 서버가 상자 위치를 계산하고 가벼운 Unreliable RPC로 양쪽 화면에 계속 동기화한다.
- 기존 씬과 상자 프리팹에 `NetworkObject`를 추가하지 않아 현재 배치를 그대로 유지했다.
- 오프라인 플레이에서는 기존 로컬 상자 들기 로직을 그대로 사용한다.

## 확인

- Unity Console 컴파일 오류 0개.
- `Assembly-CSharp.csproj` 빌드 오류 0개이며 기존 Unity 패키지 참조 경고 2개만 존재한다.
- 사용자가 편집 중인 `CharacterSelectScene`과 새 애니메이션 에셋은 수정하지 않았다.

## 빠른 테스트

1. 호스트와 클라이언트가 인게임에 입장한다.
2. 거북이가 상자 근처에서 `E`를 눌러 양쪽 화면에서 동시에 들리는지 확인한다.
3. 이동 후 다시 `E`를 눌러 같은 위치에 내려놓이는지 확인한다.

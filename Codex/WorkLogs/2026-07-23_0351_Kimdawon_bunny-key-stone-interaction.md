# 토끼 열쇠·열쇠돌 상호작용 작업

## 요청 내용

- 현재 씬의 `Key`를 토끼가 획득할 수 있게 한다.
- 열쇠를 획득한 토끼가 `KeyStone`에 접근하면 열쇠돌이 사라지게 한다.
- 기존 캐릭터 스크립트와 현재 씬의 사용자 변경 사항은 수정하지 않는다.

## 구현 내용

- `BunnyKeyStoneInteraction.cs`를 새로 추가했다.
- 실행 시 이름이 정확히 `Key`, `KeyStone`인 오브젝트가 함께 있는 씬에서 상호작용 관리 오브젝트를 자동 생성한다.
- 네트워크 플레이에서는 선택 역할이 `Bunny`인 `DualPlayNetworkPlayer`만 상호작용 대상으로 찾는다.
- 일반 씬 실행 시에는 이름이 `Bunny`인 기존 씬 오브젝트를 대체 대상으로 사용한다.
- 토끼가 `Key`에 접근하면 `Key` 오브젝트를 비활성화하고 열쇠 획득 상태를 저장한다.
- 열쇠 획득 후 토끼가 `KeyStone`에 접근하면 `KeyStone` 오브젝트 전체를 비활성화해 타일맵 렌더러와 콜라이더가 함께 사라지게 한다.
- 타일맵 오브젝트의 Transform 원점 대신 `Collider2D.ClosestPoint`를 사용해 실제 배치된 타일 가장자리와의 거리를 판정한다.
- 각 클라이언트가 동기화된 토끼 위치를 기준으로 같은 상호작용을 계산하므로 기존 네트워크 프리팹이나 캐릭터 스크립트는 변경하지 않았다.

## 확인 사항

- 현재 저장된 `InGame_1` 씬에서 `KeyStone`은 `Grid` 아래의 타일맵이며 `TilemapCollider2D`가 설정되어 있음을 확인했다.
- `Key`는 작업 시점의 저장된 씬 파일에는 아직 나타나지 않아, 하이어라키에서 이름을 `Key`로 유지하고 씬을 저장해야 한다.
- 플레이 테스트는 팀에서 진행하기로 했으므로 코드 컴파일까지만 확인한다.

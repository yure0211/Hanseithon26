# 빌드 타일맵 충돌 및 거북이 상자 들기 수정

## 확인한 원인

- `InGame_1`, `InGame_2`, `InGame_3`의 타일맵에는 활성화된 `TilemapCollider2D`와 실제 충돌 도형이 정상 생성되어 있었다.
- `Ground(6)`과 `Player(7)` 레이어 충돌도 활성화되어 있었으므로 타일맵 또는 레이어 설정 자체가 원인은 아니었다.
- 멀티플레이용 `DualPlayNetworkPlayer`가 모든 인게임 씬에서 같은 고정 시작 좌표를 사용하고 있었다.
- 이 좌표는 각 레벨에 배치된 토끼·거북이 위치와 달랐으며, `InGame_3`의 기존 토끼 좌표는 타일 충돌체와 겹쳤다.
- 거북이 상자 들기용 `TurtleCarrySkill`은 씬의 로컬 거북이에만 있고 실제 네트워크 플레이어 프리팹에는 없었다.

## 수정 내용

- 네트워크 플레이어가 인게임 씬에 들어갈 때 해당 씬에 배치된 로컬 `Bunny` 또는 `Turtle` 위치를 역할별 시작점으로 사용하도록 변경했다.
- 기존 고정 좌표는 씬에 기준 캐릭터가 없을 때만 예비 좌표로 사용한다.
- 플레이어 물리를 활성화할 때 `Dynamic Rigidbody2D`, 비트리거 콜라이더, `Player` 레이어와 `Ground` 충돌을 다시 보장한다.
- 빠른 이동에서 타일 경계를 통과하지 않도록 네트워크 플레이어 Rigidbody2D의 충돌 감지를 `Continuous`로 변경했다.
- `DualPlayNetworkPlayer` 프리팹에 `TurtleCarrySkill`을 저장했다.
- 상자 들기 스킬은 소유 중인 거북이 역할에서만 활성화하고 토끼·원격 플레이어에서는 비활성화한다.

## 확인

- 네트워크 플레이어 프리팹에 `TurtleCarrySkill` 직렬화 참조가 존재하는 것을 확인했다.
- 세 인게임 씬의 타일맵 콜라이더 활성 상태와 충돌 도형 생성을 Unity에서 확인했다.
- Unity Console 컴파일 오류 0개를 확인했다.
- `Assembly-CSharp.csproj` 빌드 결과 오류 0개이며 기존 Unity 패키지 참조 경고 2개만 존재한다.
- 진단용 임시 Editor 스크립트는 제거했다.

## 팀 빌드 테스트 항목

1. `InGame_1`부터 각 레벨에 진입해 토끼와 거북이가 벽·바닥 타일을 통과하지 않는지 확인한다.
2. 거북이로 상자 가까이에서 `E`를 눌러 들기와 내려놓기가 동작하는지 확인한다.
3. 포탈로 다음 레벨에 이동한 뒤에도 충돌과 상자 들기가 유지되는지 확인한다.

# 초기 기획 문서화 작업 기록

## 작업 목적

사용자가 전달한 게임 초기 기획을 팀이 함께 활용할 수 있는 기준 문서로 정리하고, 두 작업자가 Codex를 사용해 협업하기 위한 저장소 작업 규칙을 마련한다.

## 변경한 파일

- `Codex/GAME_DESIGN.md`
- `Codex/AGENTS.md`
- `Codex/WorkLogs/2026-07-22_Kimdawon_initial-planning.md`

## 핵심 변경 내용

- 토끼와 거북이의 역할, 시점 전환, 슬로 모션, 이로운 효과와 역경 구조를 항목별로 정리했다.
- 확정된 초기 구상과 아직 결정되지 않은 아이디어를 구분했다.
- 해커톤에서 먼저 검증할 최소 구현 범위와 다음 기획 우선순위를 제안으로 추가했다.
- Codex 작업 시작 절차, Git 협업, Unity 파일 관리, 검증과 완료 기준을 문서화했다.
- 매 저장소 변경 작업마다 파일명에 `Kimdawon`을 포함한 작업 기록을 남기도록 규칙을 추가했다.
- 모든 Markdown 문서를 `Codex/` 폴더 안에서 관리하도록 규칙을 추가했다.

## 확인 결과

- Markdown 문서 세 개가 모두 `Codex/` 아래에 생성된 것을 확인했다.
- 작업 기록 파일명에 `Kimdawon`이 포함된 것을 확인했다.
- 기획 원문에서 확정되지 않은 이야기와 시스템을 미확정 사항으로 분리했다.
- 문서만 변경했으므로 Unity 실행 테스트는 필요하지 않았다.

## 남은 작업 및 주의 사항

- 해커톤 데모의 목표 플레이 시간과 최종 목표를 팀에서 합의해야 한다.
- 거북이의 고유 능력과 시점 전환 입력 규칙을 구체화해야 한다.
- `Codex/AGENTS.md`는 `Codex/` 하위에 있으므로 저장소 전체에 대한 Codex 자동 지침으로 쓰려면 작업 시작 시 이 파일을 읽도록 요청해야 한다.
